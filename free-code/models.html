One file to manage modelsUn solo archivo para gestionar los modelos

Everything about which models free-code offers lives in ~/.free-code/agent/models.json.Todo lo relacionado con qué modelos ofrece free-code vive en ~/.free-code/agent/models.json.

~/.free-code/agent/models.json is the single place you edit to control the model list. It is re-read every time you open /model, so changes apply without restarting. Use it to:~/.free-code/agent/models.json es el único lugar que editas para controlar la lista de modelos. Se vuelve a leer cada vez que abres /model, así que los cambios se aplican sin reiniciar. Úsalo para:

  • Curate which models show in /model (the only whitelist) — including built-in ones.Seleccionar qué modelos se muestran en /model (la lista blanca only) — incluidos los integrados.
  • Add custom providers and models (Fireworks, proxies, local servers).Añadir proveedores y modelos personalizados (Fireworks, proxies, servidores locales).
  • Override built-in models (rename, change context window, route through a proxy).Sobrescribir modelos integrados (renombrar, cambiar la ventana de contexto, enrutar a través de un proxy).
Fresh installs start clean
On first run free-code seeds models.json with an only whitelist containing the most-used models (4 per provider) of each provider you can log in to, so /model is short out of the box instead of listing the whole factory catalog. It is only created if the file doesn't already exist — your edits are never overwritten.
Las instalaciones nuevas empiezan limpias
En la primera ejecución, free-code inicializa models.json con una lista blanca only que contiene los modelos más usados (4 por proveedor) de cada proveedor en el que puedes iniciar sesión, de modo que /model sea breve desde el principio en lugar de listar todo el catálogo de fábrica. Solo se crea si el archivo aún no existe — tus ediciones nunca se sobrescriben.

Curating the list — the only whitelistSeleccionar la lista — la lista blanca only

Show just the models you use, instead of every factory model.Muestra solo los modelos que usas, en lugar de todos los modelos de fábrica.

Add a top-level only array. When present and non-empty, /model shows only matching models:Añade un array only de nivel superior. Cuando está presente y no vacío, /model muestra solo los modelos que coincidan:

{
  "only": [
    "anthropic/claude-sonnet-4-6",
    "anthropic/claude-haiku-4-5",
    "fireworks/**",
    "lmstudio/**"
  ],
  "providers": {}
}
  • Add a model: add its provider/id (list ids with free-code --list-models).Añadir un modelo: añade su provider/id (lista los ids con free-code --list-models).
  • Remove a model: delete its line.Quitar un modelo: elimina su línea.
  • Show everything: remove the only key (or set it to []).Mostrar todo: elimina la clave only (o ponla en []).
  • Keep a whole provider: use a glob such as lmstudio/** or fireworks/**.Conservar un proveedor completo: usa un glob como lmstudio/** o fireworks/**.

Matching is lenient: each entry matches by provider/id, id, or name (case-insensitive; . - _ and spaces treated alike) or as a glob. If the whitelist accidentally matches nothing, free-code falls back to showing all models so the picker is never empty. Selecting a model explicitly with --model still works even if it isn't in only.La coincidencia es flexible: cada entrada coincide por provider/id, id o nombre (sin distinguir mayúsculas/minúsculas; . - _ y espacios se tratan igual) o como glob. Si la lista blanca no coincide accidentalmente con nada, free-code recurre a mostrar todos los modelos para que el selector nunca esté vacío. Seleccionar un modelo explícitamente con --model sigue funcionando aunque no esté en only.

Add a custom or local modelAñadir un modelo personalizado o local

Define providers and models under providers. Keys resolve from env vars, a shell command, or a literal value.Define proveedores y modelos bajo providers. Las claves se resuelven a partir de variables de entorno, un comando de shell o un valor literal.

Cloud (e.g. Fireworks) — key from an environment variable:Nube (por ejemplo, Fireworks) — clave desde una variable de entorno:

{
  "providers": {
    "fireworks": {
      "baseUrl": "https://api.fireworks.ai/inference/v1",
      "api": "openai-completions",
      "apiKey": "FIREWORKS_API_KEY",
      "authHeader": true,
      "models": [
        { "id": "accounts/fireworks/models/kimi-k2p7-code", "name": "Kimi K2.7 Code" }
      ]
    }
  }
}

Keys in one place
free-code loads ~/.free-code/agent/.env on startup, so you can keep FIREWORKS_API_KEY (and other secrets) there and reference them by name in models.json.
Claves en un solo lugar
