Read this first — what the installer does (and doesn't) do
The install script sets up the command-line app (CLI). On macOS it also installs the desktop app. The VS Code / Cursor plugin is NOT installed by the script — it's a short, separate step you do by hand afterwards (see Install the VS Code / Cursor plugin below).
Lee esto primero — qué hace (y qué no hace) el instalador
El script de instalación configura la aplicación de línea de comandos (CLI). En macOS también instala la app de escritorio. El plugin de VS Code / Cursor NO se instala con el script — es un paso corto y separado que haces a mano después (ver Instalar el plugin de VS Code / Cursor más abajo).

Install the VS Code / Cursor pluginInstalar el plugin de VS Code / Cursor

Optional. Adds a free-code chat panel inside your editor. Works the same on Linux and macOS.Opcional. Agrega un panel de chat de free-code dentro de tu editor. Funciona igual en Linux y macOS.

No terminal needed. You install the plugin from inside VS Code or Cursor by clicking a few buttons. Follow the steps below exactly — you don't need any command line.
No necesitas terminal. Instalas el plugin desde dentro de VS Code o Cursor haciendo clic en unos pocos botones. Sigue los pasos de abajo exactamente — no necesitas ninguna línea de comandos.

What you need firstLo que necesitas primero

  • VS Code or Cursor installed on your computer.VS Code o Cursor instalado en tu computadora.
  • The free-code folder you downloaded (the same one where the install script lives). Inside it there is a plugin file at:
    packages/vscode-free-code/vscode-free-code-0.66.1.vsix
    A .vsix file is just the packaged plugin.    La carpeta free-code que descargaste (la misma donde está el script de instalación). Dentro hay un archivo de plugin en:
    packages/vscode-free-code/vscode-free-code-0.66.1.vsix
    Un archivo .vsix es simplemente el plugin empaquetado.

Step 1 — Open the Extensions panelPaso 1 — Abre el panel de Extensiones

Open VS Code or Cursor. On the left side bar, click the Extensions icon (four little squares). Or press:Abre VS Code o Cursor. En la barra lateral izquierda, haz clic en el ícono de Extensiones (cuatro cuadraditos). O presiona:

  • macOS: Cmd + Shift + XmacOS: Cmd + Shift + X
  • Linux: Ctrl + Shift + XLinux: Ctrl + Shift + X

Step 2 — Choose “Install from VSIX…”Paso 2 — Elige “Install from VSIX…”

  1. At the top of the Extensions panel, find the small … (three dots) menu button and click it.En la parte superior del panel de Extensiones, busca el pequeño botón de menú … (tres puntos) y haz clic en él.
  2. In the menu that opens, click Install from VSIX…En el menú que se abre, haz clic en Install from VSIX…

Step 3 — Pick the plugin filePaso 3 — Elige el archivo del plugin

A file picker opens. Go into the free-code folder, then into packages → vscode-free-code, and select:Se abre un selector de archivos. Ve a la carpeta free-code, luego a packages → vscode-free-code, y selecciona:

Click Install. You'll see a small confirmation in the bottom-right corner.Haz clic en Install. Verás una pequeña confirmación en la esquina inferior derecha.

Code graph (optional)Grafo de código (opcional)

Adds symbol indexing and call graph navigation to the agent.Agrega indexación de símbolos y navegación del grafo de llamadas al agente.

# From the repository root
npm install -g ./packages/code-graph
# Desde la raíz del repositorio
npm install -g ./packages/code-graph

Once installed, the code_index, code_symbols, code_callers, and code_context tools become available automatically in every session.Una vez instalado, las herramientas code_index, code_symbols, code_callers, y code_context quedan disponibles automáticamente en cada sesión.

First use
Open a project and run /codeGraph-index. The agent will index all .ts / .js files and store the result in .code-graph/ at the project root.
Primer uso
Abre un proyecto y ejecuta /codeGraph-index. El agente indexará todos los archivos .ts / .js y guardará el resultado en .code-graph/ en la raíz del proyecto.

Add .code-graph/ to your .gitignore — the index is regenerated on demand and does not need to be committed.Agrega .code-graph/ a tu .gitignore — el índice se regenera bajo demanda y no necesita ser incluido en el control de versiones.
