Open Source · AI Tooling

free-code

A local-first AI coding agent for the terminal, VS Code/Cursor, and macOS.

What

free-code is a full-featured AI coding agent that reads and edits files, runs shell commands, browses the web, and calls MCP servers — all from inside your editor or terminal. It supports multiple AI providers (Anthropic, Google Vertex, Ollama, LM Studio, and more) and keeps everything local by default.

How

Built as a TypeScript monorepo with a plugin architecture. The agent ships with a multi-language code graph for deep code understanding, a local RAG knowledge base powered by vector embeddings, a built-in browser automation layer, and a layered extension system — skills, slash commands, and MCP tool integration.

Impact

Gives developers a privacy-first alternative to cloud-only coding assistants. By running entirely on the developer's machine, it eliminates concerns about sending proprietary code to third-party APIs while still delivering the productivity gains of AI pair programming.

Open Source · MCP / AI Tooling

mcp-flowise

An MCP server that connects AI agents directly to Flowise chatflows and pipelines.

What

mcp-flowise exposes Flowise chatflows and pipelines as callable MCP tools, letting any MCP-compatible AI agent invoke them as first-class actions — no custom glue code needed.

How

Built as a standard MCP server in TypeScript. It queries the Flowise API to discover available chatflows at startup and registers each one as a named tool with its own input schema. Agents call them like any other MCP tool.

Impact

Closes the gap between visual LLM pipeline builders and code-based agent frameworks. Teams can design flows in Flowise's no-code UI and immediately surface them as agent capabilities — without writing a single integration line.

agent-webhook-bridge

A webhook bridge that lets external systems trigger and interact with AI agents.

What

agent-webhook-bridge is a lightweight server that receives incoming webhooks from any external service and routes them as events to a running AI agent session. It closes the loop between real-world triggers (GitHub, Stripe, custom systems) and autonomous agent workflows.

How

An HTTP server registers named webhook endpoints with configurable authentication (shared secret or HMAC-SHA256). Incoming events are queued or delivered immediately to the agent via a well-defined event protocol. The bridge integrates natively with free-code's agent runtime.

Impact

Enables event-driven agentic workflows — an agent can now react to a GitHub push, a Slack message, or a payment event without any polling. This unlocks a new class of always-on automation that was previously only possible with heavyweight orchestration platforms.

Open Source · AI Agents

agent-mesh

A hub where operators publish AI agents and anyone can send them tasks from a simple web UI.

What

agent-mesh is the phase-1 implementation of the AgentMesh concept: an agent registry + job queue where each published agent is a Claude Code instance running on its operator's own machine, exposed through an agent-webhook-bridge hook. The hub never runs agents itself — it routes jobs and collects results.

How

A single-process Node hub (no build step, no dependencies) serves a web UI and a JSON API. Submitting a task fires the agent's webhook with a per-job callback URL; the bridge spawns the agent headless in a sandbox workdir and POSTs the result back when the run finishes. Jobs can resume previous Claude sessions, so conversations chain across tasks, and publishing/removing/configuring agents happens from the UI, gated by an admin token.

Impact

Turns a single machine's idle agent capacity into a shareable service with a complete task loop — submit → run → result — proving out the exposure layer the decentralized AgentMesh network builds on (tunnels, remote nodes and payments come next).
